Lesson 11: Argument Evaluation

What is it?

Put simply, argument evaluation is the way a function uses the arguments passed to it; that is, if the function will only get a reference to the arguments or if it's going to get its own copy of the values stored in the arguments.


Pass-by-value

In this method, the function gets its own copy of the values in the arguments.

				bool isEvenNumber(int number);    // isEvenNumber accepts one integer argument named "number"
				
				int main()
				{
				    int enteredNumber;
				    cout << "Enter a number: ";
				    cin >> enteredNumber;

				    if ( isEvenNumber(enteredNumber) == true )    // the value in "enteredNumber" is copied into the argument "number"
				        cout << enteredNumber << "is even.";
				    else
				        cout << enteredNumber << "is odd.";

				    return 0;
				}

				bool isEvenNumber(int number)
				{
				    // The value in "enteredNumber" is now also in "number"
				    if ( number == 0  || number % 2 == 0 )
				        return true;
				    else
				        return false;
				}

Pass-by-reference

In this method, the function only gets a reference to the arguments; that is, the addresses of the arguments are given instead of their actual values.
So, any modifications made to the arguments will also occur in the calling function's variables.

				void swapValues(int &num1, int &num2);

				int main()
				{
				    int first = 3, second = 5;

				    cout << "Original values" << endl
				         << "   first: " << first << endl
				         << "   second: " << second << endl << endl;

				    swapValues(value1,value2);

				    cout << "Exchanged values" << endl
				         << "   first: " << first << endl
				         << "   second: " << second << endl;

				    return 0;
				}

				void swapValues(int &num1, int &num2) // The address-of operator (&) says that only the addresses of the arguments will be used
				{
				    int temp;

				    temp = num1;
				    num1 = num2;
				    num2 = temp;

				    return;
				}

All arguments are evaluated in a left-to-right manner; that is, value1 will be passed as num1, value2 as num2, and so on.

Real-life example

Imagine that you had to e-mail a project quickly, but you are locked out of your account and you don't have the time to make a new one because the deadline's midnight and it's now 11:57 PM.
Luckily your friend's online, and you ask him or her for help.
You can either ask them to make or give you a new e-mail account (pass-by-value), or ask for their e-mail address and password so you can use their account temporarily (pass-by-reference)

Either way, you need to accomplish one goal - and that is to e-mail the project.
