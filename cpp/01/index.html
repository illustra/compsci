Lesson 1: Origins and Basics


C++ Origins: First Class

- Made by Bjarne Stroustrup in 1979 (Ph.D. Thesis)
- Simula - a language designed for simulations
- Simula 67 - first language with "object-oriented programming" paradigm
- C front: "C with Classes"
- C++ ("C plus plus"): The new "C with Classes" was born in 1983.
- The "++" in "C++" was a pun on the incremental operator "++" and the
original C language.

C code is compatible with C++ compilers, but not the other way round.

Dennis Ritchie, along with his companion Ken Lane Thompson, created the
original C language and the UNIX operating system.
Ken Thompson created BCPL (Basic Combined Programming Language),
the predecessor of the B programming language made for Bell Labs (now AT&T).

Basics

Structure of the main() function
(Text preceded by "//" are comments and are not part of the actual code.)
			// Let's borrow code from 'iostream' and 'cstdlib'
			#include <iostream>
			#include <cstdlib>
			using namespace std;
			
			int main() // The main function of the program
			{
			    // declaration statements go at the top of the code
			    int variable1 = 3;
				
			    // program statements go after declaration
			    cout << variable1 << endl;
				
			    // we're done
			    return 0; // Since main() has "int" before it, we will return an integer value
			}

How are C++ programs made?

First, a human (like you!) types the code into a program like, say, Code::Blocks.

Then, Code::Blocks will run its "compiler" that will "translate" the code you
typed into a language that your computer understands, like binary. It also does
the same thing with the files referred to by the "#include" statements at the
top of your code. (These files, like "iostream", are called headers, which you can think
about as tools to help your code run on a computer.)

Finally, to run your shiny new program, Code::Blocks will now run its "linker"
which will "link" together all the translated code (your code and the header files) into
one single application that you can run. :D
